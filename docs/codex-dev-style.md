# Codex Dev Style

When working in this repo:
- Read `docs/project-state.md` first.
- Inspect the repo and live n8n state before making changes.
- Ask the user before any scope decision or optional improvement.
- If there are multiple valid approaches, ask whether to choose option A, option B, or continue with the simplest path to finish the workflow first.
- Prefer finishing the user-facing workflow before doing non-essential refinements.
